Required Attributes - обязательные атрибуты
"event_id": "fc6d8c0c43fc4630ad850ee518f1b9d0" - ключ 32 символа, UUID
"timestamp": "2011-05-02T17:41:36Z" - время ошибки
"logger": "my.logger.name" - имя пользователя, который создал запись об ошибке
 "platform": "python" - хз почему, но так разработчики решили, что это очень важный параметр(возможно у них для каждого есть свои иконки :)
 Acceptable values are - Допустимые: as3,c,cfml,cocoa,csharp,elixir,haskell,go,groovy,java,javascript,native,node,objc,other,perl,php,python,ruby 
 exception - должен быть массивом, который содержит в себе струкутру. Обязательные для заполнения type или value 
 {mechanism: "Механизм", module: "Имя_Модуля", raw_stacktrace: "№Строки_исключения", stacktrace: None, thread_id: None, type: "Заголовок Ошибки", value: "Описание ошибки"}
 
 Optional Attributes - Опциональные аттрибуты
 "level": "warning" - по умолчанию = error. Допустимые: fatal, error, warning, info, debug
 "transaction": "/users/<username>/" - заголовок транзакции в которой произошло исключение 
 "server_name": "foo.example.com" - имя сервера
 "release": "721e41770371db95eee98ca2707686226b993eda" - название релиза
 "dist": "14G60" - название дистрибутива 
 "tags": {
    "ios_version": "4.0",
    "context": "production" } - теги. Их нужно передаваь структурой. Теги - это хорошо, Сентри индексирует их, поиск и группировка по тегам.
 "environment": "production" - Имя сценария. Допустимо два значения production or staging. 
 "modules": {
     "my.module.name": "1.0" } - модули в которых возникла ошибка. Список модулей
 "extra": {
       "my_key": 1,
       "some_other_value": "foo bar" } - подробное описание ошибки в виде произвольной структуры
    "fingerprint": [
         "myrpc",
         "POST",
         "/foo.bar" ] - массив тегов по которым будут группироваться ицинденты. Это отличный аттрибут для укрупненной группировки
